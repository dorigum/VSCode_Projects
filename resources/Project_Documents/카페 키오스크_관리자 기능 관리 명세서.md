# ☕ 카페 키오스크 관리자(Admin) 기능 관리 명세서

본 문서는 카페 키오스크 프로젝트의 관리자 시스템에 적용된 MVC 패턴과 계층형 아키텍처, 그리고 구현된 핵심 관리자 기능들을 상세히 분석하고 기록합니다.

---

## 1. 전체 아키텍처 및 계층별 역할

본 시스템은 **Layered Architecture**를 기반으로 하며, 각 계층은 엄격하게 분리된 책임을 가집니다.

### 🏛️ 계층 구조 (Layered Architecture)
- **View Layer (`view/`)**: 사용자 인터페이스(UI) 및 입출력 전담.
  - `MenuView`: 관리자 전용 메인 메뉴 및 서브 메뉴(카테고리/메뉴/회원/주문/통계) 루프 제어.
  - `EndView`: 성공 결과 및 데이터 목록(메뉴/통계/주문 상세) 시각화 출력.
  - `FailView`: 에러 메시지 및 비즈니스 예외 상황 출력.
- **Controller Layer (`controller/`)**: 요청 제어 및 흐름 관리.
  - `AdminController`: 서비스 계층 호출 및 결과에 따른 뷰 전환 전담. 모든 예외를 포착하여 일관된 에러 UI 제공.
- **Service Layer (`service/`)**: 핵심 비즈니스 로직 및 정책 강제.
  - `AdminServiceImpl`: 통계 집계, 지능형 매핑, 보안 정책(1인 관리자 등) 강제 로직 포함.
- **Repository Layer (`repository/`)**: 데이터베이스 영속성 관리.
  - JDBC 및 SQL을 통한 데이터 CRUD 및 트랜잭션 제어.

## 2. 구현된 핵심 관리자 기능

### 📁 2.1. 스마트 메뉴 및 카테고리 관리
- **카테고리 기반 옵션 템플릿(Template) 방식 (New)**:
    - **자동 복제 시스템**: 메뉴 등록 시 해당 카테고리에 설정된 기본 옵션들을 `MENU_OPTION_GROUP`에 자동으로 복제하여 등록합니다. 이를 통해 관리자는 메뉴마다 일일이 옵션을 설정할 필요 없이 카테고리 지정만으로 공통 옵션을 즉시 적용할 수 있습니다.
- **메뉴별 전용 옵션 예외 처리 및 오버라이딩 (New)**:
    - **유연한 옵션 구성**: 템플릿으로 자동 생성된 옵션 중 특정 메뉴에 불필요한 옵션(예: 에이드의 '온도')을 삭제하거나, 특정 메뉴에만 필요한 특수 옵션을 추가할 수 있는 **'메뉴별 전용 옵션 설정'** 기능을 제공합니다.
    - **지능형 로딩 정책**: 메뉴 조회 시 전용 설정(`MENU_OPTION_GROUP`)이 있으면 이를 최우선으로 적용하고, 설정이 없는 일반 메뉴의 경우에만 카테고리 기본값(`CATEGORY_OPTION_GROUP`)을 사용하는 하이브리드 조회 로직을 통해 데이터 무결성과 관리 편의성을 동시에 확보했습니다.
- **데이터 무결성 강화**:
    - **옵션 그룹 중복 방지**: 새로운 옵션 그룹을 등록할 때 기존에 존재하는 그룹명인지 실시간으로 체크하여 중복 데이터 생성을 원천 차단하는 검증 로직을 구현했습니다. (New)

### 👤 2.2. 고도화된 회원 관리 및 보안 시스템 (New)
- **회원 정보 시각화 고도화**:
    - **가입 일자 추적**: 회원 목록 조회 시 각 회원의 가입 날짜를 함께 출력하여 장기 고객 식별 및 회원 관리에 활용할 수 있도록 개선했습니다. (New)
    - **전문적인 관리 UI**: `ID | 전화번호 | 포인트 | 등급 | 가입일자` 구조의 표 형식 출력을 도입하여 수많은 회원 정보를 한눈에 파악할 수 있도록 디자인을 최적화했습니다. (New)
- **1인 관리자 정책 (Singleton Admin)**:
    - 시스템 안정성을 위해 관리자(`ADMIN`) 권한을 가진 계정은 오직 **단 1명**만 존재하도록 제한합니다.
    - 새로운 회원을 관리자로 승격시키려 할 때 기존 관리자가 있으면 승인을 거부하는 검증 로직이 작동합니다.

### 🎁 2.3. 투명한 포인트 관리 시스템 (Point History) (New)
- **포인트 내역 시각화 개선**:
    - 포인트 변동 히스토리를 조회할 때 `일시 | 변동금액 | 사유` 형태의 표 형식으로 출력하여 데이터의 흐름을 명확하게 표현했습니다. (New)
    - 내역이 없는 신규 회원의 경우 안내 메시지를 출력하여 사용자 경험을 개선했습니다.
- **수정 사유 필수 기록**:
    - 특정 회원에게 포인트를 지급하거나 차감할 때, 관리자는 반드시 **'수정 사유'**(예: "생일 축하 포인트🎁", "잘못된 적립 취소")를 입력해야 합니다.

### 📦 2.5. 직관적인 주문 및 취소 관리 (UX 개선)
- **실시간 주문 상세 옵션 확인**: 
    - 주문 내역 조회 시 사용자가 주문 시점에 선택했던 **실제 세부 옵션(예: 에스프레소 샷 추가, ICE 등)**을 DB에서 실시간으로 JOIN하여 출력합니다. 
    - 이를 통해 관리자는 사용자의 개별 커스텀 요구사항을 정확히 파악하여 상품을 준비할 수 있습니다. (New)
- **상세 주문 내역 조회**: 
    - 주문 상태(`[정상결제]`, `[취소완료]`), 정확한 주문 시간, 주문자 정보는 물론, 사용자가 선택한 세부 옵션 정보까지 계층형(`▶`)으로 출력하여 가독성을 극대화했습니다. (New)

## 3. 업데이트 기록 (History)

### [2026-03-17] 대규모 데이터 대응 매출 통계 엔진 및 시각화 고도화
- **지능형 통계 그래프 엔진 (Smart Scaling Graph)**:
    - **상대적 비율 시스템**: 조회된 매출 데이터 내에서 최대값을 찾아내어 그래프의 기준점으로 삼는 알고리즘을 도입했습니다. 이를 통해 수백만 원 단위의 매출 데이터가 입력되어도 그래프가 꽉 차지 않고, 데이터 간의 미세한 차이를 시각적으로 명확하게 구분할 수 있습니다. (UI 변별력 극대화)
- **최신 트렌드 우선 조회 정책 (Latest Data First)**:
    - **데이터 선별 로직**: 무의미한 과거 데이터 대신 관리자가 가장 필요로 하는 **'최근 10개 기간'**의 데이터를 우선적으로 분석하여 제공합니다.
    - **타임라인 정렬 최적화**: 최신 데이터를 추출하면서도 출력 시에는 시간의 흐름(과거 → 현재)에 따른 선형적 분석이 가능하도록 정렬 구조를 개선했습니다.
- **보고서 레이아웃 정밀화**:
    - 날짜별/기간별 통계 출력 시 라벨 정렬과 바(Bar) 출력 형식을 다듬어 전문적인 대시보드 느낌의 텍스트 UI를 완성했습니다.
    - **CSV 엑셀 호환성 원상복구**: 엑셀의 셀 분리 오류를 방지하기 위해 보고서 내 모든 수치 데이터(금액, 개수)의 쉼표(`,`) 구분자를 제거하고 순수 숫자 형식으로 출력하도록 조정하여 데이터 무결성을 확보했습니다.
- **시스템 성능 및 가시성 고도화**:
    - **핵심 로직 무결성 확보**: 신규 회원가입 시 비밀번호 암호화 불일치로 인한 포인트 적립 누락 버그를 최종 수정하여, 회원 보상 시스템의 신뢰성을 확보했습니다.
    - **데이터 로딩 최적화**:
 전체 주문 및 회원 조회 시 데이터가 많아질수록 느려지는 문제를 해결하기 위해 최신 10건 제한(`LIMIT 10`) 및 최적화된 인덱스 정렬 쿼리를 도입했습니다.
    - **관리자 우선 노출 정책**: 회원 관리 화면에서 데이터가 제한되더라도 관리 시스템 운영을 위해 `ADMIN` 역할을 가진 계정은 항상 목록 최상단에 고정 노출되도록 정렬 알고리즘을 개선했습니다.
    - **전역적 시각화 표준 적용**: 콘솔 화면의 모든 금액 데이터에 천 단위 콤마와 단위 표시를 적용하여 직관적인 데이터 분석 환경을 구축했습니다.
