# ☕ 카페 키오스크 관리자(Admin) 기능 관리 명세서

본 문서는 카페 키오스크 프로젝트의 관리자 시스템에 적용된 MVC 패턴과 계층형 아키텍처, 그리고 구현된 핵심 관리자 기능들을 상세히 분석하고 기록합니다.

---

## 1. 전체 아키텍처 및 계층별 역할

본 시스템은 **Layered Architecture**를 기반으로 하며, 각 계층은 엄격하게 분리된 책임을 가집니다.

### 🏛️ 계층 구조 (Layered Architecture)
- **View Layer (`view/`)**: 사용자 인터페이스(UI) 및 입출력 전담.
  - `MenuView`: 관리자 전용 메인 메뉴 및 서브 메뉴(카테고리/메뉴/회원/주문/통계) 루프 제어.
  - `EndView`: 성공 결과 및 데이터 목록(메뉴/통계/주문 상세) 시각화 출력.
  - `FailView`: 에러 메시지 및 비즈니스 예외 상황 출력.
- **Controller Layer (`controller/`)**: 요청 제어 및 흐름 관리.
  - `AdminController`: 서비스 계층 호출 및 결과에 따른 뷰 전환 전담. 모든 예외를 포착하여 일관된 에러 UI 제공.
- **Service Layer (`service/`)**: 핵심 비즈니스 로직 및 정책 강제.
  - `AdminServiceImpl`: 통계 집계, 지능형 매핑, 보안 정책(1인 관리자 등) 강제 로직 포함.
- **Repository Layer (`repository/`)**: 데이터베이스 영속성 관리.
  - JDBC 및 SQL을 통한 데이터 CRUD 및 트랜잭션 제어.

## 2. 구현된 핵심 관리자 기능

### 📁 2.1. 스마트 메뉴 및 카테고리 관리
- **메뉴 정보 수정 (Update)**:
    - 등록된 메뉴의 카테고리 이동, 이름 변경, 가격 조정, 상세 설명 수정을 지원하여 상품 운영의 유연성을 확보했습니다.
- **인터랙티브 옵션 설정 프로세스 (New)**:
    - **등록 후 즉시 매핑**: 메뉴 등록이 완료되면 시스템이 즉시 세부 옵션 설정 여부를 묻고, 관리자가 즉석에서 옵션 그룹을 연결할 수 있는 '연속 작업 모드'를 제공합니다.
    - 이는 상품마다 옵션 요구사항이 다른 **논커피(Non-Coffee)** 메뉴군의 관리 효율성을 극대화합니다.
- **직관적인 ID 기반 조작 UI**:
    - 수정, 삭제, 옵션 매핑 시 현재 등록된 메뉴 목록을 상단에 자동 출력하여, 관리자가 별도의 조회 과정 없이 [ID 번호]를 보고 즉시 조작할 수 있도록 개선했습니다.
- **지능형 메뉴-옵션 자동 매핑 (고도화)**:
    - 신규 메뉴 등록 시 이름에 특정 키워드(**'라떼', '프라푸치노'**)가 포함되면, 시스템이 자동으로 **'사이즈'와 '휘핑유무'** 옵션 그룹을 검색하여 즉시 연결합니다.
- **카테고리별 디폴트 옵션 정책**:
    - **커피**: 사이즈, 온도, 샷 추가, 시럽 추가, 카페인유무를 기본 제공하여 디테일한 커스터마이징 가능.
    - **논커피**: 사이즈, 온도, 퍼스널 옵션을 기본 제공 (에이드 등 논커피류의 불필요한 옵션 노출 차단).
    - **디저트**: 옵션 그룹 매핑을 제거하여 선택 과정의 번거로움 해소.
- **카테고리 기반 옵션 일괄 설정 및 자동 상속 (New)**:
    - 개별 메뉴가 아닌 카테고리 단위로 옵션 그룹을 매핑하여 운영 효율성을 높였습니다.
    - **Fallback Policy**: 특정 메뉴에 전용 옵션이 설정되지 않았더라도, 시스템이 자동으로 상위 카테고리의 옵션 규칙을 찾아 적용함으로써 관리자의 설정 누락을 방지하고 사용자에게 일관된 주문 경험을 제공합니다.

### 👤 2.2. 고도화된 회원 관리 및 보안 시스템 (New)
- **1인 관리자 정책 (Singleton Admin)**:
    - 시스템 안정성을 위해 관리자(`ADMIN`) 권한을 가진 계정은 오직 **단 1명**만 존재하도록 제한합니다.
    - 새로운 회원을 관리자로 승격시키려 할 때 기존 관리자가 있으면 승인을 거부하는 검증 로직이 작동합니다.
- **관리자 계정 삭제 보호**:
    - 관리자 계정은 실수로 삭제되는 것을 방지하기 위해 **직접 삭제가 차단**되어 있습니다.
    - 삭제를 원할 경우 반드시 먼저 '일반 회원'으로 등급을 변경한 후에만 가능합니다.
- **포인트 수정 보호**:
    - 관리자 등급 본인의 포인트는 수정할 수 없도록 제한하여 데이터 무결성을 확보했습니다.

### 🎁 2.3. 투명한 포인트 관리 시스템 (Point History) (New)
- **수정 사유 필수 기록**:
    - 특정 회원에게 포인트를 지급하거나 차감할 때, 관리자는 반드시 **'수정 사유'**(예: "생일 축하 포인트🎁", "잘못된 적립 취소")를 입력해야 합니다.
- **이력 추적**:
    - 모든 포인트 변동 내역은 사유와 함께 `POINT_HISTORY` 테이블에 기록되어, 나중에 관리자와 회원 모두가 확인할 수 있습니다.
- **테이블 자동 복구 시스템 (New)**:
    - 관리 데이터의 안정성을 위해 `POINT_HISTORY` 테이블이 삭제되거나 누락된 경우, 관리자가 데이터를 저장하거나 조회하는 시점에 시스템이 자동으로 테이블 구조를 복구(Auto-Create)하는 기능을 구현했습니다.

### 📊 2.4. 비즈니스 통계 및 데이터 리포팅 (고도화)
- **피크타임 통합 분석 (시간+요일)**:
    - 시간대별 매출 분포와 요일별 매출 현황을 한 번에 비교 분석할 수 있는 통합 리포트를 제공하여 효율적인 매장 운영을 지원합니다.
- **데이터 내보내기 (Export to CSV)**:
    - 분석된 모든 통계(일별/카테고리별/시간별/요일별/VVIP)를 CSV 파일로 추출할 수 있습니다.
    - 추출된 파일은 엑셀에서 즉시 활용 가능하며, 프로젝트 내 `resources/cafe_sales_report/` 폴더에 안전하게 자동 보관됩니다.
- **VVIP 회원 분석**: 
    - 누적 결제액 기준 상위 기여 회원을 추출하여 마케팅 지표를 제공합니다.

### 📦 2.5. 직관적인 주문 및 취소 관리 (UX 개선)
- **상세 주문 내역 조회**: 
    - 주문 상태(`[V]`, `[X]`), **정확한 주문 시간**, 주문자 정보는 물론, 사용자가 선택한 **세부 옵션 정보**까지 계층형으로 출력하여 가독성을 극대화했습니다.
- **포인트 자동 정합성 처리**: 
    - 주문 취소 시 사용 포인트 반환 및 적립 포인트 회수를 트랜잭션으로 처리하여 데이터 무결성을 보장합니다.
- **실시간 포인트 히스토리 연동 (New)**:
    - 주문 및 취소 시 발생하는 포인트 변동 사항이 즉시 히스토리 테이블에 동기화되어 관리자가 별도의 작업 없이도 전체 포인트 흐름을 한눈에 파악할 수 있습니다.
